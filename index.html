Feature Pyramid Network

发表于 2017-12-12 | 分类于 Multi-scale Object Detection

Abstract

本文根据卷积神经网络内在固有提取多尺度的特征特性，提出应用金字塔层级结构，构造卷积神经网络特征金字塔，同时控制非常小的额外时间开销。自顶向下的结构加上侧边连接用于组合不同尺度下的高层语义特征，也就是本文提出的 Feature Pyramid Network(FPN)。该算法在gpu上测试速度为6fps，在速度和准确性上都有非常好的表现。

Introduction

  1. 特征金字塔之前由图像金字塔提取手工特征得到，预测时需要使用每一层的特征
  2. 最近的检测系统使用ConvNets，对于尺度不变性表现更加鲁棒，在预测时只需要使用最高层特征
  3. 2中使用的结构对于较小的目标检测效果不是特别好，所以在预测时，使用不同卷积层的特征进行预测
  4. 3中的结构为了避免使用底层特征，ssd放弃使用之前计算的卷积层，重新添加新的卷积层，这样失去对高分辨率的特征层的重用。同时为了平衡不同层级的特征之间语义信息的差别，我们添加top-down和横跨链接

作者提出的 FPN 能够进行end-to-end的训练，能够有效提升检测和分割的准确性。

Related work

top:属于金字塔形的结构，在进行预测时还需要使用不同层级的特征，提高对不同尺度的目标识别准确性。

Multi-Context Attention for Human Pose Estimation

发表于 2017-11-23 | 分类于 human pose estimation

Abstract

本篇论文提出将multi-context attention的机制集成到卷积神经网络中，用于人体骨架关键点检测。主要使用 stacked hourglass网络从不同分辨率和不同语义的特征中产生attention map(关注图)。

本文的主要贡献：

  • 提出使用 视觉注意机制（visual attention mechanism）学习和推断上下文表示，关注感兴趣的区域；相比于传统注意机制模型中使用空间softmax归一化，本文使用CRF对相邻的关节点空间相似性进行建模；
  • 使用多重上下文注意机制（multi-context attention）使模型更加鲁棒和准确，本文使用在每一个 hourglass中使用multi-resolution ，不同的hourglass之间实现multi-semantic，最后使用 hierarchical visual attention 来聚焦于局部区域看的更清楚；
  • 新设计一个通用的 hourglass residual unit, 即在单元内部扩展一个大的图像感知域的分支，因此不同尺度的特征能够被很好的学习。

Related Work

Covolution Pose Machine

Framework

  • Baseline Network: 8-stack hourglass network
  • Nested Hourglass Network: 使用HRU结合多分辨率下的特征
  • Multi-Resolution Attention: 在每个hourglass中，多分辨率的关注图从不同尺度的图像中产生
  • Multi-Semantic Attention: 低层stack关注具体表象，高层stack编码全局的表现；所以从不同stacks中产生的attention maps也学习到不同语义的信息
  • Hierarchical Attention Mechanism: 1-4层stack，使用2个全局关注图来编码整个行人的特征，在高层，5-8层我们设计精细化的主意机制到更加局部的部位

Stacked Hourglass Networks for Human Pose Estimation

发表于 2017-11-19 | 分类于 human pose estimation

Abstract

本文主要设计一种新颖的网络结构，在不同尺度上获取特征，并整合特征获得更好的骨架信息。”stacked hourglass”网络结构由 bottom-up 和 top-down的两种结构，同时增加网络训练的中间层监督。本文实验在FLIC和MPII数据集上训练和测试，获得当时最好结果。

Introduction

目标：输入图片，得到图片中人的骨架关键点。

贡献：设计一个网络能够获取并整合不同尺度下的图片信息特征，能够获取像 人的头部，肘部这样局部的特征，同时也能够获取整个人的骨架整体的特征。

Related Work

  1. DeepPose: cmu团队Tompson的算法，输入不同尺度的图片并行处理，得到最好的检测见过；
  2. 本文设计思路起源于fully convolution networks，conv-deconv, encoder-decoder架构；

Network Architecture

Hourglass Design

动机：需要获取图像不同尺度下的特征，局部的特征用来确定头部，肘部，腰部等关键点，同时也需要获取全局的特征来整合不同部位为一个人的整体；

优点：能够很好地获得不同尺度下的特征同时也很有效地整合；

结构：convolution and max pooling 层将图像feature降到一个非常低的维度；在每个最大池化层之后，开始分支，使用在最大池化层之前相同维度的卷积；在达到最低维时，开始上采样和结合不同尺度下的特征，使用最近邻上采样，再与相同维度的特征进行element相加；在达到网络的输出部分，还有两轮1x1卷积，最终产生网络的输出。

Layer Implementation

(作者在实验室中，参考对比使用，residual learning modules and “inception”-based designs)

在保持整个hourglass形状不变，在每一层的实现上可以有很大的灵活性

作者最后选择使用residual modules, filters 不超过 3x3，这样能够控制每一层参数内存的占用；

  1. 256x256 与 7x7 ，步长为2的卷积层卷机，再加上一个 residual module 和max_pooling层，缩小至 64x64
  2. 在接下两个 residual modules
  3. 最后整个 hourglass 的所有 residual modules 输出都是 256的特征（需要好好了解resnet 结构）

Stacked Hourglass with Intermediate Supervision

蓝色为网络输出的heatmaps, 计算与ground truth的loss；之后与1x1的卷积层进行卷积，得到与中间层特征相同的维度，这些与之前的hourgalss 得到的特征相加在一起输入下一个hourglass；

注意：

  1. 网络最终使用8个 hourglass modules
  2. 每一个 hourglass modules 之间的参数是不共享的
  3. 每一个 hourglass modules 使用的 ground truth是一致的

Training Details

数据集：

  • flic: 5003(3987 训练, 1016 测试)图片;
  • mpii:25k图片, 40k 标注行人（28k训练，11k测试）

输入图片大小：256x256

数据增强：

  • 旋转： ＋－30度
  • 尺度变化：.75-1.25
  • 没有增加平移：因为检测关键点，对于图像中心点和图像本身的尺度有关，所以没有进行平移

训练环境：3天，在12g的titanx上；

测试：单张图片75ms；

Learning Feature Pyramids for Human Pose Estimation

发表于 2017-11-12 | 分类于 human pose estimation

Abstract

问题的难点：

  1. 当摄像机视角变话或者视角发生严重缩短，导致图像虚化 带来的人体骨架部位的尺度变化；
  2. pyramid methods经常在使用在inference阶段对图像进行尺度改变，在DCNNs中学习特征pyramids还未被开发

本篇文章重点：

  1. 设计 PRMs(Pyramid Residual Module) 来捕捉 DCNN 中尺度的变化，PRMs会从 不同尺度的输入特征 学习不同的 卷积核；
  2. 对于 multi－branch 网络，探索一个新的初始化参数的方法；
  3. 残差会随着深度的增加，损失会急剧增大，本文也提到一个高效的方法解决这个问题

Frame Work

PRM 提起feature －－－》stack hourglass network 获得人体骨架关键点

Revisiting Stacking Hourglass Network

Hourglass network 用于在 前向阶段 图片不同尺度的信息；bottom-up：subsampling 特征图，top-down:上采样特征图；在bottom－up 和 top－down两个阶段重复多次构成 “stacked hourglass“” 网络结构；

但是在原论文中，残差单元作为hourglass network的构建模块(积木)，只能捕捉一个scale下的可视化模式或者语义信息。本文使用的pyramid residual module作为 hourglass network的构建模块（积木）能够获取多尺度下的可视化模式或语义信息。
