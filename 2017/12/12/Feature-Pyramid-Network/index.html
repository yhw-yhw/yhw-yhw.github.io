Feature Pyramid Network

发表于 2017-12-12 | 分类于 Multi-scale Object Detection

Abstract

本文根据卷积神经网络内在固有提取多尺度的特征特性，提出应用金字塔层级结构，构造卷积神经网络特征金字塔，同时控制非常小的额外时间开销。自顶向下的结构加上侧边连接用于组合不同尺度下的高层语义特征，也就是本文提出的 Feature Pyramid Network(FPN)。该算法在gpu上测试速度为6fps，在速度和准确性上都有非常好的表现。

Introduction

  1. 特征金字塔之前由图像金字塔提取手工特征得到，预测时需要使用每一层的特征
  2. 最近的检测系统使用ConvNets，对于尺度不变性表现更加鲁棒，在预测时只需要使用最高层特征
  3. 2中使用的结构对于较小的目标检测效果不是特别好，所以在预测时，使用不同卷积层的特征进行预测
  4. 3中的结构为了避免使用底层特征，ssd放弃使用之前计算的卷积层，重新添加新的卷积层，这样失去对高分辨率的特征层的重用。同时为了平衡不同层级的特征之间语义信息的差别，我们添加top-down和横跨链接

作者提出的 FPN 能够进行end-to-end的训练，能够有效提升检测和分割的准确性。

Related work

top:属于金字塔形的结构，在进行预测时还需要使用不同层级的特征，提高对不同尺度的目标识别准确性。

Frame work

输入任意单尺度图片，输出成比例的不同尺度的特征图。该方法是独立于主要卷积架构。

bottom-up way

bottom-up就是ConvNet骨干网络的前向计算过程。

使用ResNets为骨干网络，我们使用c2，c3，c4，c5的输出，对应的步长为4，8，16，32，不使用c1，因为会占用很大的内存。

top-down way and lateral connection

top-down: 将小分辨率的特征图上采样（最近邻）

lateral：将上采样的特征图与对应的bottom-up的特征图（经过一个1x1的卷积减少通道数）进行单个元素相加

文中使用固定的特征维度为d＝256.

Application

for rpn

原始：在最后一层的特征图上，以不同大小和不同长宽比的方式在每个位置采样，一组成为一个anchor boxes

替换：使用 fpn 替换最后一层的 feature map，为了anchors 具有不同的尺度大小，通过使用相同大小的候选框在不同层级特征图上采样，{32x32 , 64x64 , 128 x128, 256x256 , 512x512 } 对应于在 {P2, P3, P4, P5, P6}上采样 ，同时在不同的层级使用三种长宽比

for fast r-cnn

原始： 使用 RoIPooling在最后一个特征层上提取特征

替换： 需要使用对应层级不同尺度的RoI，在faster-rcnn-resnet中，k0=4，尺度越小，使用越底层的特征图

Experiment

faster-rcnn baseline 中，roi pooling 输出的大小为 14x14；2fc输出的大小为7x7；所以改进的框架具有更加轻量的头部（分类器和回归模型）。

Conclusion

文中提出使用内部ConvNets构建特征金字塔。能够有效提高卷积神经网络对于多尺度的鲁棒性，而且不需要额外的计算开销；
