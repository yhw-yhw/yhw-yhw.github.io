MOT : Multi-Object Tracking

发表于 2017-10-24 | 分类于 MOT

abstract

寻找各项比赛，需要考虑fps，accuracy，使用什么样的方法（pdf，code）,各项比赛的evaluation
寻找有效的dataset，跟踪目标主要是：车，人脸，人，目标是能够达到在跟踪7～10人能够达到30fps；

为什么深度学习对于跟踪目标不好：跟踪10个苹果，而在现实场景中，跟踪物 人和车的相似度经常很相似

mot challenge（2016）

dataset

2015 年数据集的缺陷：

  • 标注协议不一致因为有些标注是直接从其它已经标注好的数据集中选择出来的；
  • 人群密度对于 训练集和测试集没有平衡
  • 有一些数据集过于简单（PETS09-S2L1），方法容易在这上 overfitting，所以这是不合理的训练集
  • 给定的检测方法在数据集上表现不够

相比于2015 年数据集的优势：

  • 给定14个数据集序列，更加的密集，从不同视角，运动相机，不同天气条件
  • 另外这些标注都被严格检验两遍，所所以标注有很高的准确性
  • 标注的类别也丰富，人，车，遮挡物，干扰物等，能够更加方便来计算被遮挡的程度

标注的规则

关注的目标：

数据集序列

  • 运动或静止的摄像头
  • 摄像头视点不同
  • 天气条件：晴天会有阴影，夜晚会运动模糊，多云则在这些方面更好；
  • 新的数据集 较之前的测试集 有三倍的 bounding boxes，人群密度也是三倍，画面质量更高

dataset： vot 2015

从 otb（51个序列） ， alov（315个序列），ptr 以及其它数据集整合到 总共有443 个序列，对这个数据集和作过滤，除去相似重复序列，以及目标面积小于400个像素的序列，最终得到356个序列；

序列属性

  • 光照明改变
  • 目标大小改变
  • 目标运动
  • 杂乱
  • 相机运动
  • 模糊
  • 目标包围盒长宽比变化
  • 目标颜色改变
  • 仿射变换
  • 场景复杂度（也就是用图像像素复杂度衡量）
  • 绝对运动；

衡量方法

  • accuracy: 通过 overlap ratio 来衡量
  • robustness: 通过 跟踪器跟踪目标丢失的次数来衡量，在这里 当overlap ratio 为0时判断跟踪目标丢失，为了更好的计算 accuracy，目标丢失后5帧会重新初始化跟踪器，重新跟踪后的目标后10帧不会用于计算准确度；
  • speed : efo（equivalent filtering operation）等价分类器运算次数，大约200 efo ＝140 fps
  • expected average overlap measure：

evaluation:

  • robustness and accuracy
  • new primary performance measure: expected average overlap on ns frames long sequences
